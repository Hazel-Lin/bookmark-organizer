# 做了一个给 AI Agent 用的书签整理 CLI

最近我把自己一团乱的 Chrome 书签，整理成了一个可复用的开源 CLI。

不是“好看一点”的那种整理。
是那种 agent 也能直接调用的整理。

它现在能做 4 件事：

1. `scan`
自动扫描 Chrome 书签里的顶层文件夹，告诉你每个文件夹有多少书签。

2. `analyze`
把目标文件夹展平。
不管你里面套了几层子文件夹，最后都会还原成一份可分析的 JSON。

3. `plan`
按 taxonomy 生成分类计划。
先 dry-run，不直接改你的书签。

4. `apply`
确认没问题后再写回，而且会自动备份。

我这次做它，核心不是“整理书签”。
而是我发现一个更大的问题：

很多 agent 工具都很强，
但一碰到这种“本地文件 + 半结构化数据 + 要求安全写回”的任务，
就容易卡在最后一步。

所以我把这件事抽成了一个标准流程：

- 先扫描
- 再展平
- 再生成 plan
- 最后 apply

这样别的 agent 不需要猜你书签结构，
只要遵守这个协议就能安全工作。

这个项目适合几类人：

- Chrome 书签已经乱掉的人
- 想把书签整理流程交给 AI 的人
- 正在做 agent-native CLI 工具的人
- 想研究“可审计自动化”怎么落地的人

我觉得这里最有意思的一点不是分类本身，
而是它天然适合做成 agent 的标准接口。

不是让 AI 直接乱改你的数据，
而是让 AI 先给你一份 plan。
